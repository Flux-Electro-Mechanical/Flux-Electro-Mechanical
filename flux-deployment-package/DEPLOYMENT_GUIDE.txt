Flux Electro Mechanical - Deployment Guide

This package is prepared for deployment.

Added deployment files:
- wsgi.py
- Procfile
- runtime.txt
- render.yaml
- updated requirements.txt with gunicorn
- .gitignore

Recommended deployment options
1. Render
2. Railway
3. VPS with Nginx + Gunicorn
4. cPanel Python App

Quick deploy on Render
1. Create a GitHub repo and upload this project.
2. In Render, create a new Web Service from the repo.
3. Render can read render.yaml automatically, or you can set:
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn wsgi:app
4. Set environment variables:
   FLUX_ADMIN_PASSWORD=your-strong-password
   PUBLIC_BASE_URL=https://your-live-domain.com
   SMTP_HOST=...
   SMTP_PORT=587
   SMTP_USERNAME=...
   SMTP_PASSWORD=...
   SMTP_USE_TLS=true
   MAIL_FROM=...
   MAIL_TO=...
5. Deploy.

Important notes
- SQLite works for testing and light usage, but for serious production use PostgreSQL is better.
- uploads, member_files, and project_images are local folders. Some cloud hosts have ephemeral storage.
- For production, migrate file storage to Cloudinary, S3, Backblaze, or similar if needed.

Local run
1. pip install -r requirements.txt
2. python app.py

Production run
1. pip install -r requirements.txt
2. gunicorn wsgi:app
